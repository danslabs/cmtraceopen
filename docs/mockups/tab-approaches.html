Approach A: Workspace Dropdown + Dedicated Tab Strip

Recommended

Workspace buttons collapse into a single dropdown. A new tab strip appears below the toolbar (only in Log workspace). Tabs show filename + close X, with an overflow dropdown for excess tabs.

Most scalable for many files Familiar VS Code / browser pattern Clean separation of concerns Adds one row of vertical space

Approach B: Workspace Dropdown + Tabs Embedded in Toolbar

Same workspace dropdown, but file tabs live inside the toolbar row \u2014 in the space freed up by removing workspace buttons. No extra row added.

No extra vertical space Compact single-row header Tabs compete with other toolbar controls Very limited tab width at 5+ files

Approach C: Unified Combo Bar

A single bar below the toolbar combines workspace selector on the left with file tabs on the right. Workspace buttons are removed from the toolbar entirely.

Compact \u2014 one row for both concepts Workspace + tabs always visible together Mixes two concepts in one bar Less space for tabs than Approach A
